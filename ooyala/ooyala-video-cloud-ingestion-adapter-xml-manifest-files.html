Ooyala-Video Cloud XML Manifest Files : Fichiers Manifest XML

Vous pouvez continuer à utiliser FTP, Aspera ou MRSS avec un manifeste XML pour ingérer vos vidéos dans Video Cloud. Cette rubrique explique la structure du XML.
Remarque : ce sujet s'applique seul aux clients d'Ooyala qui ont migré vers Video Cloud.

Introduction

Lors de l'ingestion de vidéos, vous pouvez utiliser des fichiers manifeste XML pour spécifier des métadonnées supplémentaires (y compris le titre de la vidéo, la description, les heures de vol, les étiquettes et les métadonnées personnalisées), ainsi que des fichiers de vignettes et de sous-titres codés. Vous pouvez utiliser des fichiers manifeste XML lorsque vous ingérez du contenu avec FTP, Aspera ou MRSS.

Note : Cette rubrique s'applique à OIS v2.5.

Gestion des métadonnées

Si vous incluez metadata des vidéos lorsque vous les ingérez, elles sont mappées aux champs vidéo Video Cloud de l'une des deux manières suivantes :

  1. Par défaut, chaque paire metadata clé/valeur est ajoutée en tant que étiquette sous la forme : meta::key::value. Par exemple, <ooyala:metadata name="category">cooking</ooyala:metadata> produirait la balise meta::category::cooking
  2. Si vous préférez les utiliser custom_fields pour stocker des métadonnées, le support Brightcove peut modifier un paramètre de votre compte pour ce faire. Soyez conscient de ce qui suit :
    • Vous devrez créer des champs personnalisés avec des noms correspondant à vos clés de métadonnées.
    • Le nombre de champs personnalisés est limité à 10 par défaut. Par conséquent, si vous avez besoin de plus que cela pour accommoder vos métadonnées, vous devrez demander des champs personnalisés supplémentaires auprès du Support.
    • Si un champ personnalisé correspondant à chaque élément de métadonnées n'existe pas dans votre compte, toute l'opération de métadonnées échouera.

Éléments du manifeste XML

Vous pouvez utiliser les éléments suivants dans le fichier manifeste XML.

Éléments du manifeste XML
Balise XML Description Obligat/Facultatif
<channel> Contient un ou plusieurs <item> éléments. requis
<item> Représente une seule vidéo à ingérer. Vous pouvez spécifier plusieurs vidéos (<item> balises) dans un fichier manifeste XML. Contient les balises restantes dans ce tableau. requis
<média : content> Spécifie une vidéo à ingérer. L'un des éléments suivants :
  • Nom de fichier d'une vidéo que vous avez téléchargée.
  • Pour l'ingestion via MRSS (voir Ingestion à partir d'un flux MRSS hébergé à distance), l'URL où se trouve la vidéo. L'URL peut inclure http :// ou https ://.
Pour un actif distant, utilisez <ooyala : actif distant> au lieu.		 requis (sauf pour les ressources distantes)
<média : title> Nom ou titre de la vidéo. optionnel
<média : vignette > Spécifie une miniature (aperçu de l'image). L'un des éléments suivants :
  • Nom de fichier d'une miniature que vous avez téléchargée.
  • URL où se trouve la vignette. L'URL peut inclure http :// ou https ://.
 optionnel
<Médias : Sous-titre > Spécifie un fichier de sous-titres (pris en charge pour les fichiers DFXP, WebVTT et SRT). Reportez-vous à la section Ingrétion de fichiers de sous-titres. Attributs :
  • href (obligatoire) - L'un des éléments suivants :
    • Nom du fichier de sous-titres codés que vous avez téléchargé. Exemple : légende-1.dfxp
    • URL où se trouve le fichier de sous-titres. L'URL peut inclure http :// ou https ://.
    langue (facultatif) - langue du fichier de sous-titres codés. Code à deux lettres. Voir Sous-titres codés pris en charge.
  • frameRate (facultatif) - Fréquence d'images du fichier de sous-titres codés. Les unités sont exprimées en images par seconde (FPS). La valeur par défaut est 30. Les anciens fichiers de sous-titres codés peuvent être 24.
Note : Si spécifié, il est de votre responsabilité de fournir des valeurs valides pour langue et frameRate. Des valeurs valides sont ajoutées au fichier DFXP. Les valeurs non conformes sont ignorées. Quoi qu'il en soit, le fichier de sous-titres codés est ajouté dans Backlot et lié à l'actif.
 optionnel
<média : description> Description de la vidéo. optionnel
<relier> URL permanente où vous intégrez la vidéo. Correspond à l'actif créé hébergé_à biens. optionnel
<dcterms : valid> Temps de vol représentant les heures de début et de fin de lecture de l'actif. Exemple : 
  <DCTerms:Valid>Start=2011-06-28T 15:15:00-07:00 ; Fin = 2022-12-01T 16:00:00-07:00 ; Schéma = W3C-DTF </dcterms:valide >
 optionnel
<ooyala : étiquettes > Une ou plusieurs étiquettes, séparées par des virgules. Exemple : /sports, /sports/jogging. Pour en savoir plus, voir Étiquettes. optionnel
<media : mots-clés> Une autre façon de définir des étiquettes (sans barre oblique). Le contenu, séparé par des virgules, sera ajouté en tant que balises à la vidéo. Une nouvelle balise ne sera créée que si une balise portant le même nom n'existe pas déjà.

Par exemple, pour ce qui suit <media : mots-clés> étiquette, les étiquettes étiquette 1 et étiquette 2 seraient créés et ajoutés à la vidéo en tant que balises. Une étiquette nommée étiquette parent serait également créé, avec le label étiquette enfant niché à l'intérieur.

  <media:keywords > label 1, label 2, label<parent/enfant/media:mots clés >
Note : Le contenu de <media : mots-clés> sont ignorés lorsque le <ooyala : étiquettes > est incluse dans le fichier manifeste XML ou MRSS.
 optionnel
<ooyala : code incorporé> ID vidéo Video Cloud. facultatif (obligatoire si vous mettez à jour une vidéo plutôt que d'en ajouter une nouvelle)
<ooyala : guide du profil> ID du profil de traitement, qui est un groupe d'encodages qui définissent les formats dans lesquels une vidéo principale doit être convertie lors du transcodage. Contactez votre représentant du support Ooyala pour plus de détails. optionnel
<ooyala : métadonnées> Métadonnées personnalisées pour la vidéo. Vous pouvez avoir un élément pour chaque type de métadonnées. Reportez-vous à la section Gestion des métadonnées pour plus de détails sur metadata l'ajout aux vidéos en tant que balises spéciales. Exemple : <ooyala:metadata name="category">cooking</ooyala:metadata> optionnel
<ooyala : actif distant> Spécifie une ressource distante à ingérer. Une ressource distante est une vidéo qui est hébergée ailleurs que Backlot. Spécifiez les attributs suivants :
  • duréeEnMs: durée du média (en millisecondes)
  • url: URL de la ressource distante. L'URL peut inclure http :// ou https ://.
Ce paramètre correspond à la valeur du hébergé_à propriété pour un actif distant. Exemple : <ooyala : durée de l'actif distant en MS='5000' url= » http://your_website/your_remote_assets.mp4 "/>
Note : Une fois qu'une ressource distante a été créée, sa propagation vers les différents CDN peut être retardée de 60 secondes ou plus. Si vous demandez une ressource distante trop tôt après sa création, les résultats seront mis en cache par les CDN, ce qui peut prendre plusieurs minutes pour effacer. Meilleure pratique : après la création, patientez 30 ou 60 secondes, interrogez avec l'API Backlot [GET] /v2/assets/asset_id route, et après avoir récupéré le code d'intégration de l'actif distant (ID de contenu ou ID d'actif), puis procédez à l'intégration de l'actif.
 requis (ressources distantes uniquement)
<ooyala : guide du profil> Spécifie un profil d'ingestion Video Cloud à utiliser pour le traitement de la vidéo. Si vous ne souhaitez pas utiliser le profil par défaut du compte, vous devez spécifier celui que vous souhaitez utiliser à la place. Voir Gestion des profils d'ingest. optionnel

Étape par étape

Création d'un fichier manifeste XML

Pour créer un fichier de métadonnées XML :

  1. Téléchargez vos vidéos et autres contenus (miniatures, fichiers de sous-titres codés, etc.).
  2. Ouvrez un éditeur de texte, de préférence celui qui valide XML.
  3. Ajoutez les en-têtes XML requis. Par exemple : 
      <? xml version="1.0" encoding="utf-8" ? > version <rss ="2.0" xmlns:media= » http://search.yahoo.com/mrss/ "xmlns:dcterms= » http://purl.org/dc/terms/ "xmlns:fh= » http://purl.org/syndication/history/1.0 "xmlns:ooyala= » http://www.ooyala.com/mrss/ « >
  4. Créer un canaliser récipient.
  5. Créé un article entrée pour chaque vidéo, contenant les éléments souhaités du tableau ci-dessus.

    Dans l'exemple suivant, XML spécifie les métadonnées pour deux vidéos. Pour la première vidéo, le fichier et les vignettes se trouvent sur un serveur de téléchargement et sont retirés de ce serveur. Pour la deuxième vidéo, le fichier et la vignette ont déjà été téléchargés sur un serveur Ooyala. La deuxième vidéo a également des temps de vol.

      <? xml version="1.0" encoding="utf-8" ? > version <rss ="2.0" xmlns:media= » http://search.yahoo.com/mrss/ "xmlns:dcterms= » http://purl.org/dc/terms/ "xmlns:fh= » http://purl.org/syndication/history/1.0 "xmlns:ooyala= » http://www.ooyala.com/mrss/ « > <cannel> <item> <media:content url= » http://mysite.com/upload/lacrosse_70.mov "/> <media:thumbnail url= » http://mysite.com/upload/lacrosse_70_previewimage.jpg "/> <media:title>Sports qui existent vraiment : Crosse</media:title > <Média:Description>Ma description de ma<vidéo/média:description > <ooyala:labels>/sports/crosse,/ hobbies/crosse</ooyala:labels > <lien> http://mysite.com/videos/sports/real_sports.html</link > <ooyala:metadata name = » video_id"> 70</ooyala:metadata > <ooyala : nom de métadonnées = « saison_number » > 1</ooyala : métadonnées > </item > <item> <media:content url= » curling_71.mov "/> <media:miniatures url= » curling_71_previewimage.jpg "/> <Media:TITLE>Sports qui existent vraiment : Curling</media:title > <Media:Description>Ma description de ma<vidéo/média:description > <ooyala:labels>/sports/curling,/ hobbies/curling</ooyala:labels > <lien> http://mysite.com/videos/sports/real_sports.html</link > <ooyala:metadata name = » video_id">71</ooyala : métadonnées > <ooyala : nom de métadonnées = "saison_number">1</ooyala : métadonnées > < — définir les heures de vol —> <DCTerms:Valid>Start=2011-06-28T 15:15:00-07:00 ; End=2022-12-01T 16:00:00-07:00 ; Scheme=W3C-DTF </dcterms:valid > </item > </canal > </rss >
  6. Enregistrez le fichier au format XML. Veillez à spécifier un nom unique. Si vous spécifiez deux fichiers portant le même nom et téléchargez l'un pendant que l'autre est en cours de traitement, le second peut être ignoré.
  7. Avec la méthode de téléchargement de votre choix, téléchargez le fichier manifeste XML lorsque vous avez téléchargé la ou les vidéos. Voir soit FTP ou Aspera.
