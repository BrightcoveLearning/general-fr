Ooyala vers Video Cloud Migration : Ajout du serveur DNS Brightcove pour les tests

Afin de tester l'adaptateur de lecture et le connecteur CMS qui vous permettent de continuer à utiliser les lecteurs Ooyala pour diffuser du contenu Video Cloud, vous devrez ajouter un serveur DNS à vos systèmes et fournir des informations à Brightcove pour la configuration. Cette rubrique vous guide à travers les étapes.

Remarque : ce sujet s'applique seul aux clients Ooyala migrés vers Video Cloud.

Introduction

À test l'adaptateur de lecture qui permettra à vos lecteurs Ooyala de diffuser du contenu à partir de Video Cloud et/ou d'utiliser le connecteur CMS qui vous permet d'utiliser certaines opérations du API Backlot pour gérer vos médias dans Video Cloud, deux choses doivent se produire :

  1. Tout d'abord : contactez votre CXM pour lancer le processus. Votre CXM devra obtenir des adresses IP auprès de vous et vous aidera à les trouver si vous ne savez pas comment le faire. Votre CXM peut également vous aider avec les étapes décrites dans le reste de ce document que vous devez effectuer.
  2. Ce que vous devez faire pour chaque ordinateur et appareil qui utilisera l'adaptateur de lecture ou le connecteur CMS est d'ajouter un serveur DNS Brightcove qui redirigera les demandes envoyées aux ressources ooyala.com vers les API Brightcove appropriées. Les étapes à suivre sont détaillées dans les sections qui suivent.
Notes :
  • Sachez que si vous modifiez le DNS sur un ordinateur ou un périphérique avant que son adresse IP ne soit mise en liste blanche par votre CXM, il peut désactiver l'accès à Internet.
  • Le serveur DNS supplémentaire n'est requis que pour les tests. Une fois que vous avez vérifié que vous pouvez utiliser les lecteurs et services Ooyala dont vous avez besoin avec les données Video Cloud, vous pouvez supprimer le serveur DNS Brightcove des ordinateurs et appareils que vous utilisez pour tester.

Étapes macOS

En supposant que vous n'êtes pas sur un VPN, voici les étapes pour ajouter le serveur DNS Brightcove sur macOS pour ordinateurs portables et de bureau Mac.

  1. Ouvrez les Préférences Système :
  2. Ouvrez les Préférences Réseau :
  3. Clique sur le Avancée Les paramètres:
  4. Ouvrez les paramètres DNS :
  5. Clique le + signer pour ajouter un serveur DNS :
  6. Saisissez l'adresse suivante pour le nouveau serveur DNS :

    35.170.73.174

  7. Clique le + connectez-vous sous Domaines de recherche pour ajouter un domaine de recherche :
  8. Saisissez le domaine de recherche suivant :

    ooyala.com

  9. Cliquez sur d'accord pour fermer les paramètres avancés.
  10. Cliquez sur Appliquer pour enregistrer les nouveaux paramètres.
  11. Redémarrez votre Mac pour effacer le cache DNS.

Étapes de Windows 10

Voici les étapes à suivre pour ajouter le serveur DNS Brightcove sous Windows 10 pour les ordinateurs portables et de bureau.

  1. Ouvrez le Panneau de configuration.
  2. Cliquez sur Réseau et Internet.
  3. Cliquez sur Centre Réseau et partage.
  4. Cliquez sur l'option Modifier les paramètres de l'adaptateur dans le volet de gauche.
  5. Cliquez avec le bouton droit sur l'interface réseau connectée à Internet et sélectionnez l'option Propriétés :
  6. Sélectionnez et cochez l'option Protocole Internet version 4 (TCP/IPv4).
  7. Cliquez sur le bouton Propriétés.
  8. Cliquez sur l'option Utiliser les adresses de serveur DNS suivantes.
  9. Pour vous Préféré sur l'adresse du serveur DNS alternatif, entrez 35.170.73.174
  10. Cliquez sur OK.
  11. Cliquez sur proche
  12. Redémarrez l'ordinateur pour effacer le cache DNS.

Cas particulier : VPN

Les étapes ci-dessus peuvent ne pas fonctionner si votre ordinateur/périphérique est sur VPN, car le serveur VPN peut remplacer vos paramètres de serveur DNS par les siens. Si cela se produit, vous devrez ajouter le serveur DNS Brightcove à votre VPN. Exactement quelles commandes vous aurez besoin dépendront de la plate-forme et du VPN, mais un exemple pour macOS est montré ci-dessous pour vous donner une idée.

Obtenez la sous-clé de l'entrée DNS sur l'interface VPN. Par exemple avec Pulse Secure, il ressemble à :

sudo scutil
> get subKey [127 = State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS]
> d.add ServerAddresses * 35.170.73.174
> set State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS
