    Migration Ooyala vers Video Cloud : Ajout du serveur DNS Brightcove pour les tests

    Afin de tester l'adaptateur de lecture et le connecteur CMS qui vous permettent de continuer à utiliser les lecteurs Ooyala pour diffuser du contenu Video Cloud, vous devrez ajouter un serveur DNS à vos systèmes et fournir des informations à Brightcove pour la configuration. Cette rubrique vous guide à travers les étapes.

    Remarque : cette rubrique s'applique uniquement aux clients Ooyala migrés vers Video Cloud.

    Introduction

    Pour tester l' adaptateur de lecture qui permettra à vos lecteurs Ooyala de livrer du contenu à partir de Video Cloud, et/ou d'utiliser le connecteur CMS qui vous permet d'utiliser certaines opérations de Backlot API pour gérer vos médias dans Video Cloud, deux choses doivent se produire :

    1. Tout d'abord : contactez votre CXM pour lancer le processus. Votre CXM devra obtenir des adresses IP auprès de vous et vous aidera à les trouver si vous ne savez pas comment le faire. Votre CXM peut également vous aider avec les étapes décrites dans le reste de ce document que vous devez effectuer.
    2. Ce que vous devez faire pour chaque ordinateur et appareil qui utilisera l'adaptateur de lecture ou le connecteur CMS est d'ajouter un serveur DNS Brightcove qui redirigera les demandes envoyées aux ressources ooyala.com vers les API Brightcove appropriées. Les étapes à suivre sont détaillées dans les sections qui suivent.
    Remarques :
    • Sachez que si vous modifiez le DNS sur un ordinateur ou un périphérique avant que son adresse IP ne soit mise en liste blanche par votre CXM, il peut désactiver l'accès à Internet.
    • Le serveur DNS supplémentaire n'est requis que pour les tests. Une fois que vous avez vérifié que vous pouvez utiliser les lecteurs et services Ooyala dont vous avez besoin avec les données Video Cloud, vous pouvez supprimer le serveur DNS Brightcove des ordinateurs et appareils que vous utilisez pour tester.

    Étapes macOS

    En supposant que vous n'êtes pas sur un VPN, voici les étapes pour ajouter le serveur DNS Brightcove sur macOS pour ordinateurs portables et de bureau Mac.

    1. Open System Preferences:
    2. Open the Network Preferences:
    3. Click On the Advanced settings:
    4. Open the DNS settings:
    5. Click the + sign to add a DNS server:
    6. Enter the following address for the new DNS server:

      35.170.73.174

    7. Click the + sign under Search Domains to add a search domain:
    8. Enter the following search domain:

      ooyala.com

    9. Click OK to close the Advanced settings.
    10. Click Apply to save the new settings.
    11. Restart your Mac to clear the DNS cache.

    Étapes de Windows 10

    Voici les étapes à suivre pour ajouter le serveur DNS Brightcove sous Windows 10 pour les ordinateurs portables et de bureau.

    1. Open Control Panel.
    2. Click on Network and Internet.
    3. Click on Network and Sharing Center.
    4. Click the Change adapter settings option in the left pane.
    5. Right-click the network interface connected to the internet, and select the Properties option:
    6. Select and check the Internet Protocol Version 4 (TCP/IPv4) option.
    7. Click the Properties button.
    8. Click the Use the following DNS server addresses option.
    9. For you Preferred on Alternate DNS server address, enter 35.170.73.174
    10. Click OK.
    11. Click Close
    12. Restart the computer to clear the DNS cache.

    Étapes iOS

    Voici les étapes à suivre pour ajouter le serveur DNS Brightcove sur iOS pour iPhone et iPad.

    Notez que l'appareil doit se trouver sur une connexion Wi-Fi pour effectuer cette modification.

    1. Open Settings:
    2. Open the Wi-Fi settings:
    3. Open the I (information) for the current network:
    4. Tap Configure DNS to change the configuration:
    5. Change the DNS configuration from Automatic to Manual
    6. Add a DNS server:
    7. For the new server address, enter 35.170.73.174
    8. Save your settings.
    9. Restart the device to clear the DNS cache.

    Étapes Android

    En raison des différentes versions d'Android et des différentes façons de modifier le serveur DNS selon que vous disposez d'un accès root à l'appareil, et si vous utilisez des paramètres système ou une application tierce, nous vous recommandons de contacter votre CXM pour obtenir de l'aide pour effectuer cette modification sur Android. Si vous préférez essayer vous-même d'abord, vous pouvez essayer les instructions ici.

    Cas particulier : VPN

    Les étapes ci-dessus peuvent ne pas fonctionner si votre ordinateur/périphérique est sur VPN, car le serveur VPN peut remplacer vos paramètres de serveur DNS par les siens. Si cela se produit, vous devrez ajouter le serveur DNS Brightcove à votre VPN. Exactement quelles commandes vous aurez besoin dépendront de la plate-forme et du VPN, mais un exemple pour macOS est montré ci-dessous pour vous donner une idée.

    Obtenez la sous-clé de l'entrée DNS sur l'interface VPN. Par exemple avec Pulse Secure, il ressemble à :

    sudo scutil
> get subKey [127 = State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS]
> d.add ServerAddresses * 35.170.73.174
> set State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS
