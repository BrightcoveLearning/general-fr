Introduction

Utilisez le API de backlot pour télécharger par programme du contenu dans Video Cloud. Cette approche vous permet d'intégrer votre système de gestion de contenu (CMS) ou vos flux de travail directement à la plateforme Video Cloud.

Notez que vous pouvez également migrer vos intégrations ou applications vers le API CMS et API d'ingestion dynamique à tout moment.

Vous pouvez utiliser le API de backlot pour télécharger et gérer par programmation des vidéos, des vignettes et des fichiers de sous-titres codés.

Si vous souhaitez ingérer du contenu à l'aide de l'API Backlot, consultez :

  • Ressources vidéo et audio (pour les vidéos et les miniatures)
  • Utilisation des sous-titres codés

L'appel principal utilisé pour télécharger des fichiers (appelés « actifs ») est le /v2/actifs route.

Note : Ne pas ajouter de vidéo avec un null external_id, c'est-à-dire un external_id sans valeur ("") ou une valeur de "nul". De tels ID externes null ne peuvent pas être recherchés ultérieurement.

Ressources vidéo et audio

Lorsque vous téléchargez une vidéo sur Ooyala, Ooyala la transcode automatiquement en utilisant les paramètres spécifiés pour votre compte et rend les vidéos disponibles pour la lecture via votre lecteur.

Notez que les ressources audio seront stockées dans Video Cloud sous forme de vidéos sans piste vidéo.

Nous vous recommandons de télécharger des vidéos sur Ooyala dans H.264, mais nous prenons également en charge de nombreux codecs couramment utilisés.

Pour créer de nouvelles ressources vidéo ou audio :
  1. Créer l'actif avec des propriétés de base
  2. Récupérer les « URL de téléchargement » du système
  3. Pour chaque « URL de téléchargement », mettez le morceau de fichier approprié sur le système
    Note : Lorsque vous téléchargez chaque morceau, ne signez pas la demande, comme vous le feriez pour d'autres demandes. Il suffit de mettre le contenu du bloc aux URL exactes. Ces URL ont déjà été signées pour vous.

Obtenir le téléchargement des URL pour les ressources nouvellement créées

    [GET]/v2/assets/{asset_id}/uploading_urls
Note : Lorsque vous téléchargez chaque morceau, ne signez pas la demande, comme vous le feriez pour d'autres demandes. Il suffit de mettre le contenu du bloc aux URL exactes. Ces URL ont déjà été signées pour vous.

Définir le statut de chargement

    [PUT] /v2/assets/{asset_id}/upload_status { "status":"uploaded" }

Afficher les flux d'une vidéo

Le terme ruisseaux dans ce contexte, signifie les processus internes à Ooyala qui sont lancés par le système pour transcoder vos vidéos.

Note : Vous ne pouvez pas contrôler ou manipuler ces flux ; ils sont contrôlés par le système. Vous ne pouvez voir que leurs noms, pas les changer.
    [GET]/v2/assets/{asset_id}/ruisseaux

Afficher les fichiers source d'une vidéo

    [GET]/v2/assets/{asset_id}/source_file_info

Obtenir les URL pour les images d'aperçu générées automatiquement

    [GET] /v2/assets/{asset_id}/generated_preview_images

Télécharger une image d'aperçu personnalisée

    [POST]/v2/actifs/{asset_id}/preview_image_files< contenu_fichier>

Définir la configuration d'image d'aperçu principal

Définissez le type de l'image d'aperçu principale d'une ressource sur l'une des options suivantes :
  • généré: utiliser l'image d'aperçu générée automatiquement
  • fichier_téléchargé: utilisez l'image d'aperçu personnalisé téléchargée
  • remote_url: URL de l'image d'aperçu
    [PUT]/v2/assets/{asset_id}/primary_preview_image { "type":"generated" | "uploaded_file" | "remote_url" }

A propos des mét

Un actif peut avoir jusqu'à 100 paires nom/valeur de métadonnées personnalisées. Un nom est limité à 128 caractères et sa valeur est limitée à 2 048 caractères.

Exemples

Cet exemple crée une vidéo : 
    [POST]/v2/éléments{ "name":"Honey Badger", "file_name":"honeybadger.avi", "asset_type":"video", "file_size":398422, "chunk_size":100000 }

Video Cloud renvoie une réponse similaire à la suivante :

    { "name":"Honey Badger", "preview_image_url":null, "asset_type":"video", "duration":0, "embed_code":{embed_code}, « created_at » :"2011-10-26T 19:40:32 + 00:00 «, « time_restrictions » :null, « updated_at » :"2011-10-26T 19:40:32 + 00:00 «, « external_id » :null, « hosted_at » :null, « original_file_name » : » honeybadger.avi «, » ; description » :null, « status » :"uploading »}

Cet exemple renvoie les URL de téléchargement :

    [GET]/v2/assets/{asset_id}/uploading_urls

Cet exemple montre comment marquer la vidéo comme téléchargée, ce qui indique à Video Cloud de commencer à traiter le fichier :

    [PUT]/v2/assets/{asset_id}/upload_status { "status":"uploaded" }

Note : Les URL des fichiers source sont signées et expirent dans les 24 heures.

Utilisation des sous-titres codés

Vous pouvez ajouter des sous-titres codés à des vidéos ou à des vidéos de ressources distantes via l'API Backlot.

Formats supportés pour les sous-titres

Formats supportés
Extension de fichier Description
.vtt (WebVTT/) - c'est le format requis pour les lecteurs vidéo HTML5 WebVTT peut être utilisé pour les sous-titres embarqués et sidecar dans des formats de streaming HTTP tels que HLS et DASH.
.TTML (Auparavant DXFP) - sera converti en WebVTT TTML (Fichier de langage de balisage de texte chronométré. En règle générale, un fichier de sous-titres codés de ce type aura une extension .dfxp (profil d'échange de format de distribution).

Charger les sous-titres

    [PUT]/v2/assets/{asset_id}/sous-titres_fermés/{dfxp_file_content}

Obtenir des sous-titres

    [GET]/v2/assets/{asset_id}/sous-titres_fermés

Mettre à jour les sous-titres

    [PUT]/v2/assets/{asset_id}/sous-titres_fermés

Supprimer les sous-titres

    [SUPPRIMER]/v2/actifs/{asset_id}/sous-titres_fermés
