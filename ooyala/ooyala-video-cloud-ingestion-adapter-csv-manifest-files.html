    Ooyala-Video Cloud Ingestion Adaptateur : Fichiers Manifest CSV

    Vous pouvez continuer à utiliser FTP ou Aspera avec un manifeste CSV pour ingérer vos vidéos dans Video Cloud. Cette rubrique explique la structure du CSV.
    Remarque : cette rubrique s'applique uniquement aux clients Ooyala migrés vers Video Cloud.

    Introduction

    Lors de l'ingestion de vidéos, vous pouvez utiliser des fichiers manifeste CSV pour spécifier des métadonnées supplémentaires (y compris le titre de la vidéo, la description, les heures de vol, les étiquettes et les métadonnées personnalisées), ainsi que des miniatures et des fichiers de sous-titres codés. Vous pouvez utiliser des fichiers manifestes CSV lorsque vous ingérez du contenu avec FTP ou Aspera.

    Contraintes

    Note : Cette rubrique s'applique à OIS v2.5.

    Le format CSV du fichier de métadonnées prend en charge moins de fonctionnalités que le format XML. Le format CSV vous permet de spécifier des miniatures (aperçu des images) et des métadonnées personnalisées à associer à une vidéo. Le format XML prend en charge ces fonctionnalités ainsi que l'association des noms des jeux d'annonces définis existants.

    Gestion des métadonnées

    Si vous incluez metadata des vidéos lorsque vous les ingérez, elles sont mappées aux champs vidéo Video Cloud de l'une des deux manières suivantes :

    1. Par défaut, chaque paire metadata clé/valeur est ajoutée en tant que balise sous la forme : meta::key::value. Par exemple, <ooyala:metadata name="category">cooking</ooyala:metadata> produirait la balise meta::category::cooking
    2. Si vous préférez les utiliser custom_fields pour stocker des métadonnées, le support Brightcove peut modifier un paramètre de votre compte pour ce faire. Soyez conscient de ce qui suit :
      • Vous devrez créer des champs personnalisés avec des noms correspondant à vos clés de métadonnées.
      • Le nombre de champs personnalisés est limité à 10 par défaut. Par conséquent, si vous avez besoin de plus que cela pour accommoder vos métadonnées, vous devrez demander des champs personnalisés supplémentaires auprès du Support.
      • Si un champ personnalisé correspondant à chaque élément de métadonnées n'existe pas dans votre compte, toute l'opération de métadonnées échouera.

    En-têtes de colonne du manifeste CSV

    Les champs suivants sont la première ligne (en-têtes de colonne) de votre fichier CSV, avec les champs de données réels suivant les lignes suivantes (une ligne par vidéo).

    Note : Vous pouvez spécifier ces colonnes dans n'importe quel ordre, et vous pouvez omettre les colonnes dont vous n'avez pas besoin, à condition que les données des lignes de données correspondent exactement aux en-têtes.
    En-têtes de colonne du manifeste CSV
    Entête de colonne Description Obligat/Facultatif
    video Nom de fichier de la vidéo ou de l'URL où se trouve la vidéo. L'URL peut inclure http:// ou https://. requis
    title Nom ou titre de la vidéo. optionnel
    thumbnail Nom de fichier de la miniature ou de l'URL où se trouve la vignette. optionnel
    content_type Type de contenu (vidéo ou ressource distante). Si le nom de fichier spécifié dans la colonne vidéo est une URL où se trouve la vidéo et si vous souhaitez définir une ressource distante, modifiez la valeur de la colonne content_type (de video, valeur par défaut) par content_type=remoteasset. requis (ressources distantes uniquement)
    description Description de la vidéo. optionnel
    hosted_at URL permanente où vous intégrez la vidéo. Mappe à la valeur de la hosted_at propriété pour une ressource distante.
    Note : Une fois qu'une ressource distante a été créée, sa propagation vers les différents CDN peut être retardée de 60 secondes ou plus. Si vous demandez une ressource distante trop tôt après sa création, les résultats seront mis en cache par les CDN, ce qui peut prendre plusieurs minutes pour effacer. Bonne pratique: après la création, attendez 30 ou 60 secondes, interrogez avec le API Backlot [GET] /v2/assets/asset_id et après avoir récupéré le code d'intégration de l'actif distant (ID de contenu ou ID d'actif), procédez à l'incorporation de l'actif.
    		 optionnel
    flight_start_time Heure de début à laquelle l'actif peut être joué, en UTC. Exemple : 2011-06-01T00:00:00Z obligatoire (seulement si elle flight_end_timeest spécifiée)
    flight_end_time Heure de fin de lecture de l'actif, en UTC. Exemple : 2011-07-01T00:00:00Z optionnel
    durationInMs (OS v2.5 uniquement) Durée représentant la longueur de la vidéo (en millisecondes). Requis quand content_type=remoteasset. requis (uniquement pour les ressources distantes)
    labels Une ou plusieurs étiquettes, séparées par des virgules. Par exemple : /sports,/sports/jogging. optionnel
    metadata Métadonnées personnalisées pour la vidéo. Vous pouvez avoir une colonne pour chaque type de métadonnées. Consultez la section Gestion des métadonnées ci-dessus pour voir comment les métadonnées sont mappées aux champs vidéo Video Cloud. optionnel
    embed_code (Réservé). Content ID dans Backlot de la ressource vidéo. réservé
    id Mappe à la propriété ID externe de la ressource créée : un identificateur personnalisé que vous définissez que vous pouvez utiliser à la place de l'ID de contenu. optionnel
    subTitle Spécifie un fichier de sous-titres codés (pris en charge pour les fichiers DFXP uniquement). L'un des éléments suivants :
    • Nom du fichier de sous-titres codés que vous avez téléchargé. Exemple : <media:subTitle href="caption-1.dfxp"/>
    • URL où se trouve le fichier de sous-titres. L'URL peut inclure http:// ou https://.

    Reportez-vous à la section Ingrétion de fichiers de sous-titres.

    		 optionnel
    subTitle:lang (facultatif) - Langue du fichier de sous-titres codés. Code de langue à deux lettres./td > optionnel
    subTitle:frameRate (facultatif) - Fréquence d'images du fichier de sous-titres codés. Les unités sont exprimées en images par seconde (FPS). La valeur par défaut est 30. Les anciens fichiers de sous-titres codés peuvent être 24. optionnel
    profileguid ID du profil de traitement, qui est un groupe d'encodages qui définissent les formats dans lesquels une vidéo principale doit être convertie lors du transcodage. Si vous ne souhaitez pas utiliser le profil d'ingeste Video Cloud par défaut, vous devez inclure l'ID de celui que vous souhaitez utiliser. Voir Gestion des profils d'ingest. optionnel

    Exemple de fichier manifeste CSV (OIS v2.5)

    Création d'un fichier manifeste CSV

    Gardez à l'esprit les points suivants lors de la génération d'un fichier de métadonnées CSV :

    • Si une valeur de texte dans un champ de données comporte une virgule (,), vous devez placer le champ entre guillemets (« ).
    • Si une valeur de texte dans un champ de données comporte un guillemet double (« ), vous devez placer le champ entre guillemets doubles et échapper au caractère de guillemets doubles dans le champ de texte avec un deuxième guillemet (par exemple, « Elle a dit, « " Avoir une belle journée » " »)

    Si vous créez votre fichier de métadonnées CSV à partir d'Excel, il gère automatiquement l'échappement.

    Pour créer un fichier de métadonnées CSV :

    1. Téléchargez une ou plusieurs vidéos.
    2. Ouvrez un programme de feuille de calcul, tel que Microsoft Excel.
    3. Ajoutez les en-têtes de colonne à la première ligne.
    4. Ajoutez une ligne pour chaque vidéo.
    5. Enregistrez le fichier au format CSV. Par exemple, vous pouvez sélectionner Enregistrer sous dans le menu Fichier . Lorsque vous y êtes invité, sélectionnez le type de fichier de valeurs séparées par des virgules.
    6. Avec la méthode de téléchargement de votre choix, téléchargez le fichier CSV lorsque vous avez téléchargé la ou les vidéos. Voir soit FTP ou Aspera.
