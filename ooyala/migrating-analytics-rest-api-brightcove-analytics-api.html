    Migration de l'API REST Analytics vers l'API Brightcove Analytics

    Cette rubrique fournit des conseils sur le passage de l'API REST Ooyala Analytics à l'API Video Cloud Analytics.
    Remarque : cette rubrique s'applique uniquement aux clients Ooyala migrés vers Video Cloud.

    Introduction

    Video Cloud Analytics API est assez similaire à l'API REST Ooyala Analytics.

    Notez également que le module Analytics de Video Cloud Studio inclut un générateur de rapports personnalisés robuste qui, dans de nombreux cas, peut rendre inutile l'utilisation de l'API.

    Les sections ci-dessous correspondent aux différents aspects des deux API :

    • L' authentification
    • Rapports
    • Paramètres
    • Dimensions
    • Filtres
    • Mesures

    Authentification

    L'authentification des requêtes API Brightcove est traitée légèrement différemment de celle des API REST Ooyala. Pour plus de détails, reportez-vous à Authentification pour les requêtes d'API Brightco

    Rapports

    Le seul /reports Le point de terminaison pour l'API REST Ooyala Analytics correspond très étroitement à Brightcove Analytics API /data point final. La principale différence est que l'API Ooyala permet à la fois GET et POST , ce dernier a utilisé pour transmettre des données dans le corps de la requête plutôt que des paramètres d'URL, ce qui permet des chaînes de données plus longues.

    Paramètres

    Les paramètres des requêtes sont similaires pour les deux API. Ils sont détaillés dans le tableau ci-dessous pour vous montrer où se situent les différences.

    aucun accounts - inclus en tant que paramètre dans tous les rapports sur plusieurs comptes Vue d'ensemble des dimensions, des champs et des paramètres
    report_type Aucune, mais l'API Brightcove offre des points de terminaison supplémentaires pour les rapports d'engagement et d'analyse en direct Vue d'ensemble : API Analytics

    end_date to les deux paramètres sont similaires et acceptent tous deux des valeurs dans le formulaire YYYY-MM-DD ; le param Brightcove accepte également les dates d'époque en millisecondes et les dates relatives telles que -3d Vue d'ensemble des dimensions, des champs et des paramètres

    aucun bucket_duration Vue d'ensemble des dimensions, des champs et des paramètres

    Dimensions :

    Les API Ooyala et Brightcove Analytics regroupent les données en compartiments appelés « dimensions ». Ils sont utilisés de manière similaire, mais les API ont des règles quelque peu différentes quant à la façon dont plusieurs dimensions peuvent être combinées, et Brightcove a un ensemble plus large de dimensions. Pour plus d'informations sur l'API Brightcove, reportez-vous à la section Combinaisons de dimensions prises Le tableau ci-dessous fournit une comparaison détaillée.

    country country - les dimensions sont essentiellement identiques Dimension Pays
    region region - les dimensions sont essentiellement identiques, sauf que la dimension Brightcove inclut les États américains Dimension de région

    Filtres

    Les filtres fonctionnent à peu près de la même manière dans les deux API, spécifiées en termes de :

          
        // Ooyala
        filter={dimension1}=={value1};{dimension2}=={value2}
      
        // Brightcove
        where={dimension1}=={value1};{dimension2}=={value2}

    Mesures

    Le metrics (pour l'API Brightcove : fields ) renvoyé dans les rapports varie en fonction de la ou des dimensions demandées pour l'API Brightcove Analytics. Vous pouvez utiliser l'outil intégré dans Dimensions et champs pour voir quels champs peuvent être renvoyés pour une dimension particulière ou une combinaison de cotes.

    Notez que bien que l'API Ooyala renvoie toutes les mesures si aucune n'est spécifiée, mais l'API Brightcove renvoie seulement un petit ensemble de mesures par défaut qui varient selon la dimension (combinaison), sauf si vous incluez le fields dans la requête.

    Les champs Brightcove contiennent des équivalents pour toutes les mesures Ooyala à l'exception de :

    • uniq_plays_requested
    • uniq_displays
    • uniq_video_starts

    Brightcove Analytics API peut également renvoyer le nombre d'autres mesures non disponibles dans Ooyala analytics.
