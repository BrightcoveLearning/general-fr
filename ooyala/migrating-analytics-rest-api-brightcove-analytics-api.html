Introduction

Le nuage vidéo API d'analyse est assez similaire à l'API REST Ooyala Analytics.

Notez également que le module Analytics de Video Cloud Studio inclut un générateur de rapports personnalisés robuste qui, dans de nombreux cas, peut rendre inutile l'utilisation de l'API.

Les sections ci-dessous correspondent aux différents aspects des deux API :

  • L' authentification
  • Rapports
  • Paramètres
  • Dimensions
  • Filtres
  • Mesures

Authentification

L'authentification des requêtes API Brightcove est traitée légèrement différemment de celle des API REST Ooyala. Voir Authentification des demandes d'API Brightcove pour plus de détails.

Rapports

Le seul /reports point de terminaison pour l'API REST Ooyala Analytics correspond très étroitement à Brightcove API d'analyse /data point final. La principale différence réside dans le fait que l'API Ooyala permet à la fois GET et POST, à cette dernière, de transmettre certaines données dans le corps de la requête plutôt que dans les paramètres d'URL, permettant d'utiliser des chaînes de données plus longues.

Dimensions :

Les API Ooyala et Brightcove Analytics regroupent les données en compartiments appelés « dimensions ». Ils sont utilisés de manière similaire, mais les API ont des règles quelque peu différentes quant à la façon dont plusieurs dimensions peuvent être combinées, et Brightcove a un ensemble plus large de dimensions. Voir Combinaisons de dimensions prises en charge pour plus de détails sur l'API Brightcove. Le tableau ci-dessous fournit une comparaison détaillée.

Filtres

Les filtres fonctionnent à peu près de la même manière dans les deux API, spécifiées en termes de :

      
        // Ooyala
        filter={dimension1}=={value1};{dimension2}=={value2}
      
        // Brightcove
        where={dimension1}=={value1};{dimension2}=={value2}

Mesures

La metrics(pour l'API Brightcove : fields) renvoyée dans les rapports varie en fonction de la ou des dimensions demandées pour l'API Brightcove Analytics. Vous pouvez utiliser l'outil intégré dans Dimensions et champs pour voir quels champs peuvent être renvoyés pour une dimension particulière ou une combinaison de dimensions.

Notez que bien que l'API Ooyala renvoie toutes les mesures si aucune n'est spécifiée, mais l'API Brightcove renvoie seulement un petit ensemble de mesures par défaut qui varient selon la dimension (combinaison), sauf si vous incluez le fields dans la requête.

Les champs Brightcove contiennent des équivalents pour toutes les mesures Ooyala à l'exception de :

  • uniq_plays_requested
  • uniq_displays
  • uniq_video_starts

Le Brightcove API d'analyse peut également renvoyer le nombre d'autres métriques non disponibles dans les analyses Ooyala.
