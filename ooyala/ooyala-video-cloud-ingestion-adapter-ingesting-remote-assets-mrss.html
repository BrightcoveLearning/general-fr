Ooyala-Video Cloud : Ingestion d'actifs distants via MRSS

Vous pouvez continuer à utiliser MRSS pour créer et mettre à jour des ressources distantes dans Video Cloud.
Remarque : ces sujets s'appliquent seul aux clients d'Ooyala qui ont migré vers Video Cloud.

Introduction

Vous pouvez ajouter ou mettre à jour des ressources distantes à l'aide de Media RSS (MRSS).

Voir Travailler avec des actifs distants pour une meilleure compréhension des actifs distants dans Video Cloud.

Lorsque vous utilisez MRSS pour ajouter ou mettre à jour un actif distant, vous identifiez l'actif comme distant en spécifiant le < oyala:remoteasset> balise (au lieu de < média:contenu> ) dans l'élément de fil MRSS.

Note : Pour décider d'ajouter ou de mettre à jour une ressource distante, Brightcove utilise le <ooyala:embedcode></ooyala:embedcode> champ du manifeste XML pour déterminer si la ressource distante existe actuellement (mise à jour) ou non (ajouter).

Ajout d'une ressource distante

L'exemple de code suivant crée une ressource distante (le < guid spécifié > n'existe pas).

<rss xmlns:media= » http://search.yahoo.com/mrss/ "xmlns:dcterms= » http://purl.org/dc/terms/ "xmlns:fh= » http://purl.org/syndication/history/1.0 "xmlns:ooyala= » http://www.ooyala.com/mrss/ "version="2.0"> <canal > <item > <ooyala : durée des actifs distants en MS='1000' url= » http://mysite.com/RCTTestAssets/mrss/assets/Test.mp4"/ > <media:thumbnail url= » http://mysite.com/test.jpg"/ > <Media:Subtitle href= » http://mysite.com/test.dfxp"/ > <Media:title> Test</media:title > <Média:Description><Test/média : description > <OOYala : Labels>/Test</ooyala:labels > <lien> http://mysite.com/videos/sports/test_0010.html</link > < — définir les heures de vol —> <dcterms:valid> start=2011-06-28T 15:15:00-07:00 ; fin = 2022-12-01T 16:00:00-07:00 ; schéma = W3C-DTF </dcterms:valid > </item > </canal > </rss >

Mise à jour d'une ressource distante

Pour une ressource distante existante, vous pouvez mettre à jour les champs suivants via MRSS :

  • média : title
  • média : description
  • Ooyala : sous-titre
  • ooyala : métadonnées
  • dcterms:valide
  • média : vignette
  • ooyala : étiquettes

Lorsque vous soumettez l'élément de flux MRSS, s'il contient des mises à jour de l'un de ces champs dans le <item> tag, le système met à jour l'actif distant avec les modifications applicables.
