Ooyala Video Cloud Feed MRSS hébergé à distance : Ingération à partir d'un flux MRSS hébergé à distance

Vous pouvez continuer à ingérer des vidéos et des ressources dans Video Cloud à l'aide des flux MRSS via l' adaptateur d'ingestion.
Remarque : ce sujet s'applique seul aux clients d'Ooyala qui ont migré vers Video Cloud.

Introduction

Vous pouvez utiliser Media RSS (MRSS) pour demander à Ooyala d'extraire votre contenu de votre propre système à partir d'un flux MRSS que vous publiez. Video Cloud extrait les métadonnées du flux MRSS et crée une ressource distante ou télécharge (à partir de l'endroit où elles sont hébergées) et traite les vidéos et tous les fichiers associés (miniatures, fichiers de sous-titres codés, etc.).

Note : Cette rubrique s'applique à OIS v2.5.

Présentation de MRSS Ingest

À propos du MRSS

Media RSS (MRSS) est une extension de RSS qui vous permet de syndiquer des fichiers multimédia dans des flux RSS. Pour plus de détails techniques, reportez-vous à la spécification RSS des médias.

Configuration de MRSS

Contactez votre gestionnaire de compte et indiquez que vous souhaitez utiliser MRSS. Pour chaque vidéo que vous souhaitez ingérer, vous devez fournir à Ooyala des URL de contenu valides. Les URL peuvent inclure http :// ou https ://.

Utilisation du MRSS

Pour utiliser MRSS, vous hébergez des métadonnées et/ou des vidéos et d'autres fichiers (miniatures, sous-titres codés) sur un serveur Web et créez un fichier manifeste XML qui fait référence à ce contenu.

Utilisation de MRSS pour extraire les métadonnées uniquement

  1. Créez un fichier manifeste XML comme décrit dans Fichier manifeste XML. Vous devez utiliser le ooyala : ressource distante balise pour l'ingestion de métadonnées uniquement.
  2. Hébergez le fichier manifeste XML sur l'un des emplacements MRSS que vous avez fournis. Assurez-vous que le nom du fichier et le chemin correspondent exactement à l'un des chemins que vous avez fournis à Ooyala.

Utilisation de MRSS pour extraire des vidéos, des miniatures et des fichiers de sous-titres codés

  1. Hébergez un ou plusieurs fichiers (vidéos, miniatures et fichiers de sous-titres codés) sur un serveur Web.
  2. Créez un fichier manifeste XML (voir Fichier manifeste XML). Assurez-vous que les entrées du fichier référenent les fichiers sur votre serveur. En particulier, la valeur de la <média : content> éléments href attribut doit être une URL HTTP vers le fichier sur votre serveur, comme dans l'exemple suivant : 
    <media:content url= » http://mysite.com/upload/lacrosse_70.mov "/>
  3. Hébergez le fichier manifeste XML sur l'un des emplacements MRSS que vous avez fournis. Assurez-vous que le nom du fichier et le chemin correspondent exactement à l'un des chemins que vous avez fournis à Ooyala.

    Note : Malheureusement, le concept d' « enregistrement des clients » ne s'applique pas à la transformation à base de MRSS. Le meilleur moyen de vérifier que votre flux est traité avec succès est de vérifier l'apparence de vos ressources, soit avec l'interface utilisateur de Video Cloud Studio, soit avec le API de backlot /v2/actifs demander.

Une fois les vidéos et tous les autres fichiers associés traités, pour extraire plus de fichiers, vous hébergez simplement un autre fichier MRSS à l'emplacement convenu.
