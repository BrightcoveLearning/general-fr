Remarque : ce sujet s'applique seul aux clients d'Ooyala qui ont migré vers Video Cloud.

Nouvelles options de Video Cloud

Video Cloud propose trois options standard pour l'ingestion de vidéos :

  • Le module Upload dans Studio
  • Ingestion basée sur des extractions à l'aide de l'API Dynamic Ingest
  • Téléchargement du fichier source à l'aide de l'API Dynamic Ingest

Ou utiliser des méthodes d'ingestion Ooyala

En tant que client Video Cloud, vous pouvez utiliser l'une des options énumérées ci-dessus, mais Brightcove a également construit des adaptateurs pour les systèmes d'ingest Ooyala pour diriger les vidéos vers le système d'ingeste Video Cloud, de sorte que vous pouvez continuer à utiliser le service d'ingestion Ooyala (OIS) pour envoyer vos vidéos à Video Cloud à l'aide de :

  • FTPS
  • Aspera
  • Les flux MRSS hébergent à distance
  • L' API Backlot

Toutes les options de manifeste existantes continueront de fonctionner. Continuez à utiliser vos identifiants d'authentification pour Ooyala, et l'adaptateur les traduira automatiquement en informations d'identification pour les API Video Cloud nécessaires :

Ingestion basée sur l'API
Si votre flux de travail d'ingestion comprend l'ingestion de nouveaux contenus directement via l'API Ingestion : lorsque votre trafic client est dirigé vers Video Cloud, vous utiliserez la clé et le secret de l'API Ooyala existants pour effectuer les appels d'API vers les terminaux actuels de l'API Ooyala.
Ingestion à base de FTP/ASpera
Assurez-vous de réinitialiser le mot de passe du compte utilisateur « Télécharger uniquement » Vidéo pourrait être différent de celui que vous aviez dans Backlot. Si votre compte « téléchargement uniquement » dans Backlot supporté à la fois la vidéo et les métadonnées, alors le compte « Télécharger uniquement » Video Cloud le fera également. Vous pouvez continuer à vous connecter aux mêmes points de terminaison FTP/ASPERA et utiliser vos nouveaux identifiants de compte utilisateur « télécharger uniquement » Video Cloud pour télécharger des fichiers. Une fois les fichiers téléchargés et disparus du dossier de téléchargement, connectez-vous à Video Cloud et assurez-vous que votre vidéo a été ajoutée. Si vous téléchargez des fichiers de métadonnées XML ou CSV avec des vidéos et que vous souhaitez cibler un profil d'ingestion spécifique, vous devez spécifier le nom du profil Video Cloud dans le champ XML ou CSV sous l'élément profileguid. Voir Gestion des profils d'ingest.
Ingestion à base de MRSS
Veuillez partager votre flux MRSS avec votre équipe de compte et elle peut l'activer. Comme nous ne pourrons pas valider quelles vidéos du flux MRSS ont été ingérées dans Backlot, assurez-vous que seules les nouvelles vidéos sont disponibles dans le flux MRSS. Si d'anciennes vidéos sont disponibles dans le flux, elles seront également ingérées, ce qui donne du contenu en double dans Video Cloud. Une fois le flux MRSS en place, vous pouvez vous connecter à Video Cloud pour valider que le contenu est ingéré.
Ingestion via Brightcove Video Cloud Studio
Une différence importante avec Backlot est le profil d'ingestion que vous choisirez pour ingérer votre contenu, assurez-vous de choisir un profil d'ingeste dans Video Cloud qui répond le mieux à vos besoins, vous trouverez des informations utiles sur les profils d'ingest Video Cloud dans Gestion des profils d'ingest.
