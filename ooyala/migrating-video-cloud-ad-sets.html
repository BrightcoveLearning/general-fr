Migration des jeux d'annonces

Video Cloud n'a pas le concept d'un jeu d'annonces. Les annonces côté client sont généralement configurées par le biais de points de cueils vidéo et d'un plugin publicitaire pour le joueur. Voir étape par étape : Mise en œuvre de la publicité pour plus de détails.

Lors de la migration de vidéos vers Video Cloud associées à un tel ensemble, nous faisons deux choses :

  • Migration de certaines données de jeu d'annonces vers des points de cueille (ce qui est migré varie selon le ad_source, et est détaillé dans les sections ci-dessous)
  • Stockage de l'identifiant du jeu d'annonces dans un champ personnalisé appelé ad_set_id, afin que la vidéo puisse être mise à jour ultérieurement pour les modifications apportées au jeu d'annonces

Si la vidéo est jouée dans un lecteur Ooyala existant, les annonces devraient continuer à être diffusées comme auparavant. Pour les lecteurs Brightcove, vous devrez ajouter et configurer le plugin publicitaire approprié au lecteur. Voir Configuration de la publicité du lecteur à l'aide du module Players et l'autre documentation publicitaire du lecteur Brightcove. pour plus de détails.

Sources publicitaires prises en charge

Seuls les ad_source types suivants sont pris en charge pour la migration des données d'ensemble d'annonces vers Video Cloud :

  • Videoplaza
  • Google IMA v3
  • Freewheel
  • Mes annonces
  • VAST
  • VPAID

Les champs suivants seront supprimés pour toutes les sources de la liste ci-dessus qui le prennent en charge :

  • tracking_pixel_url
  • ad_frequency

Les champs supplémentaires pour ces ad_sources seront stockés comme cue_points[i].metadata du type « CODE » en tant que JSON stringifié. Voir les sections ci-dessous pour les mappages exacts.
