Vocabulaire de base

Types d'annonces

  • Publicités linéaires : Les annonces qui se produisent pendant la lecture prennent du temps dans la chronologie (par exemple, un pré-roll)
  • Publicités non linéaires : N'interrompt pas la lecture ; superposition et annonces complémentaires en sont des exemples

Normes publicitaires

  • VMAP : « playlist d'annonces » ; spécifie le pré-roll, le (s) milieu (s) et le post-roll (URL d'un document VMAP) ; le format est XML

    • DFP appelle les documents VMAP « règles publicitaires » ; avec les règles publicitaires, vous pouvez configurer toutes vos annonces sur le serveur et tout client qui comprend les règles publicitaires se comporte de la même manière ; configuration publicitaire « mauvais vieux jours » au client ; logique déplacée dans le système de publication

  • VAST : Détails de l'annonce individuelle ; VMAP pointe vers un document VAST contenant des informations sur l'annonce

  • Exemple VAST à partir de l'onglet Réseau dans les outils de développement Chrome

  • VPAID : Comme VAST, mais une norme pour les annonces interactives ; l'annonce contient un quiz ou un jeu maladroit (pas encore de prise en charge de BC Player)

Informations sur le plugin IMA

  • Contrainte IMA : 2 versions de la bibliothèque IMA, version Flash ou HTML, ne peuvent pas utiliser à la fois

    • paramètres par défaut pour le plugin utiliser Flash si disponible (car les annonces Flash sont si répandues)
    • Dans les options de configuration pour le plugin peuvent basculer par défaut en HTML (AdTechOrder)
    • Flash par défaut du bureau, HTML par défaut pour mobile

  • Mise en œuvre de base du plugin IMA

    • Inclure JavaScript pour le plugin
    • Ajouter IMA Plugin au lecteur
    • Fournir ServerURL à la balise d'annonce
    • Exemple de configuration

  • Ciblage dynamique des annonces : Configuration du plugin IMA effectuée après la collecte des informations client (par exemple, âge des utilisateurs, puis sélectionnez la règle d'annonce)

    • Inclure JavaScript pour le plugin
    • Appelez le plugin IMA et les options de passe à ce stade
      • MyPlayer.IMA3 ({serverURL :...})
  • Annonces complémentaires : IMA Plugin traitera les annonces linéaires et les superpositions de manière transparente, les annonces compagnons peuvent être difficiles puisque la communication entre le joueur et le reste de la page doit se produire ; ; ; utiliser l'implémentation d'intégration dans la page
    • Utilisez la bibliothèque Google Publisher Tags (GPT) et remplira automatiquement les blocs publicitaires compagnons
    • Bon tutoriel disponible sur le site IMA
    • Utiliser l'implémentation intégrée dans la page pour les annonces compagnons car iframe ne peut pas interagir avec la page parent
    • Exemple : JS et HTML pour les annonces complémentaires
