Introduction

Les règles de livraison de Brightcove sont une fonctionnalité de la livraison dynamique qui vous permet de tirer parti de la capacité de génération de manifestes juste à temps sur une base plus granulaire. Avec les règles de livraison, vous pouvez créer des règles personnalisées pour contrôler la façon dont votre contenu est remis à l'utilisateur. Cela vous permet de diffuser du contenu de manière plus efficace et efficiente.

Les règles de remise vous permettent de créer une logique conditionnelle en fonction du type d'appareil, de l'emplacement de l'utilisateur ou d'un signal provenant de votre application. Lorsque ces conditions sont déclenchées, vous contrôlez le comportement tel que les types de formats associés renvoyés et le CDN à utiliser.

Exemples

Comme vous pouvez l'imaginer, les combinaisons de logique conditionnelle peuvent être infinies. Voici quelques exemples de ce que les règles de livraison peuvent faire :

Niveaux d'abonnement multiples

Un modèle commun pour les services de vidéo à la demande par abonnement (SVOD) consiste à offrir deux niveaux d'abonnement : définition standard (SD) et haute définition (HD).

Sur la base d'un signal de votre application, vous pouvez demander à Dynamic Delivery de ne renvoyer que les formats associés SD pour les utilisateurs standard et les formats associés HD pour les utilisateurs premium.

CDN optimisés pour la géographie

Bien que la plupart des CDN soient optimisés pour la distribution mondiale, certaines régions sont mieux desservies par un CDN spécifique à une région. Un CDN dans la région ou dans le pays peut être utilisé pour une livraison optimale.

Avec les règles de remise, vous pouvez créer une règle de sorte que lorsqu'une visionneuse se trouve dans une zone spécifique, un CDN spécifique à une région sera utilisé. Pour tous les autres utilisateurs, un CDN global sera utilisé.

Formats associés optimisés pour les périphériques

Étant donné que les consommateurs visualisent les médias à partir d'une variété de tailles d'écran, il n'est pas idéal de fournir le même ensemble de formats associés à chaque appareil.

Avec Delivery Rules, vous pouvez créer une règle qui détecte si l'appareil du visualiseur est un téléphone et exclure les rendus 1080p. Vous pouvez disposer d'une autre règle qui détecte si une visionneuse se trouve sur un périphérique OTT et exclure les formats associés inférieurs à 360p.
