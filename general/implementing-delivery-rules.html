Vue d'ensemble

Les éditeurs doivent souvent modifier les caractéristiques de contenu de leurs manifestes médias pour des raisons techniques et commerciales.

Pour répondre à ces besoins, Brightcove a créé un cadre dans lequel des règles peuvent être appliquées qui modifieront le contenu du manifeste renvoyé par Dynamic Delivery. Les comportements sont définis comme des conditions « si » qui déclencheront une certaine règle et une série de paramètres « alors » qui définiront la façon dont le manifeste est modifié.

Si vous n'êtes pas familier avec cette fonctionnalité, reportez-vous à la section Présentation : Document Règles de livraison dynamiques.

Application des règles

Les diagrammes de flux suivants montrent comment les règles de remise sont appliquées.

Comportements

Les comportements, ou conditions « si », pris en charge dans la première phase sont les suivants :

  • Demande explicite : ID de configuration que vous pouvez transmettre à l'API de lecture lors de l'exécution. Cet ID est mappé aux valeurs de configuration qui permettent aux services sous-jacents de prendre des décisions exploitées par l'API de lecture pour renvoyer les données de réponse appropriées.

  • Groupe de périphériques : type de périphérique détecté par l'analyse de l'agent utilisateur. Les groupes actuellement pris en charge sont les suivants :
    • android
    • appletv
    • iPad
    • iphone
    • ordinateur de bureau chromé
    • bureau-firefox
    • bureau-safari
    • fetchtv
    • ios-autre
    • roku-7/8
    • smartphone-générique
    • tablette-android
    • tablette-Kindle
    • inconnu
     
  • Géographie - Emplacement physique de l'appareil demandeur :
    • Continent
    • Pays

Règles

Les règles, ou paramètres « alors », pris en charge dans la première phase sont les suivants :

  • CDN Media Delivery - À partir des CDN configurés pour être utilisés dans un compte Dynamic Delivery donné, le CDN à utiliser pour la livraison de segments de médias.

  • Caractéristiques du rendu : filtre en fonction des caractéristiques des rendus multimédias sous-jacents. Il s'agit notamment de :
    • Débit vidéo minimum
    • Débit vidéo maximal
    • Résolution vidéo minimale
    • Résolution vidéo maximale
    • Nombre maximal de formats associés audio
    • Nombre maximal de formats associés vidéo
    • Premier débit vidéo
    • Discontinuités

Portée

Les règles peuvent être appliquées à deux niveaux :

  • Compte - Règles qui s'appliquent à toutes les demandes adressées au compte d'un client spécifique
  • Demande spécifique : règles invoquées sur une demande spécifique

Les règles seront appliquées dans l'ordre indiqué ci-dessus. Les règles ultérieures remplaceront les précédentes.

Activation de la fonction

Au cours de la phase initiale des règles de livraison, toutes les règles seront configurées par Brightcove. Contactez votre gestionnaire de compte pour en savoir plus sur la tarification et activer cette fonctionnalité pour votre compte.

Contraintes

Il existe quelques limitations connues lors de l'utilisation des règles de livraison :

  • Contenu multimédia - Les règles de diffusion ne fonctionnent pas avec les vidéos Smooth ou PMP4.
  • Audio uniquement - Les règles de diffusion avec les publicités côté serveur (SSAI) activées nécessitent du contenu vidéo et audio. Il s'agit d'une restriction SSAI.
