A

API
interface de programmation d'applications - interface informatique qui définit les interactions entre plusieurs intermédiaires logiciels. Il définit les types d'appels ou de requêtes qui peuvent être effectués, comment les faire, les formats de données à utiliser, les conventions à suivre, etc. Une API peut être entièrement personnalisée, spécifique à un composant, ou elle peut être conçue en fonction d'une norme de l'industrie pour garantir l'interopérabilité.

C

CDN
Réseau de diffusion de contenu - réseau géographiquement distribué de serveurs proxy et de leurs centres de données. L'objectif est de fournir une haute disponibilité et des performances en distribuant le service spatialement par rapport aux utilisateurs finaux. Les CDN sont apparus à la fin des années 1990 pour atténuer les goulots d'étranglement de performance d'Internet.
CMS
système de gestion de contenu - logiciel informatique utilisé pour gérer la création et la modification de contenus numériques. Il existe de nombreux SMC Web, comme Wordpress et Drupal, utilisés pour gérer le contenu d'un site Web, mais les SGC peuvent être utilisés pour gérer tout type de contenu. Par exemple, la médiathèque Video Cloud est un CMS pour le contenu multimédia et les ressources multimédias, et c'est pourquoi l'API principale utilisée pour gérer ce contenu est appelée API CMS.

D

TIRET
Dynamic Adaptive Streaming sur HTTP - une technique de streaming adaptative qui permet une diffusion de contenu multimédia de haute qualité sur Internet à partir de serveurs Web HTTP conventionnels. Semblable à la solution HTTP Live Streaming (HLS) d'Apple, MPEG-DASH fonctionne en divisant le contenu en une séquence de petits segments, qui sont servis sur HTTP.
Gestion des droits numériques (DRM)
Gestion des droits numériques - les outils ou les mesures de protection technologique sont un ensemble de technologies de contrôle d'accès destinées à restreindre l'utilisation de matériel propriétaire et d'œuvres protégées par le droit d'auteur. Les technologies DRM tentent de contrôler l'utilisation, la modification et la distribution d'œuvres protégées par le droit d'auteur (telles que les logiciels et le contenu multimédia), ainsi que des systèmes au sein d'appareils qui appliquent ces politiques.

H

HLS
HTTP Live Streaming - un protocole de communication en streaming adaptatif basé sur HTTP développé par Apple Inc. et publié en 2009. La prise en charge du protocole est très répandue dans les lecteurs multimédias, les navigateurs Web, les appareils mobiles et les serveurs multimédia en streaming.
HTTP
Hypertext Transfer Protocol - protocole de couche applicative pour les systèmes d'information hypermédia distribués, collaboratifs. HTTP fonctionne comme un protocole de demande—réponse dans le modèle informatique client—serveur. Un navigateur Web, par exemple, peut être le client et une application exécutée sur un ordinateur hébergeant un site Web peut être le serveur. Le client envoie un message de requête HTTP au serveur. Le serveur, qui fournit des ressources telles que des fichiers HTML et d'autres contenus, ou exécute d'autres fonctions pour le compte du client, renvoie un message de réponse au client. La réponse contient des informations sur l'état d'achèvement de la demande et peut également contenir du contenu demandé dans le corps du message.

M

MP4
MPEG-4 Part 14 - un format de conteneur multimédia numérique le plus couramment utilisé pour stocker de la vidéo et de l'audio, mais il peut également être utilisé pour stocker d'autres données telles que des sous-titres et des images fixes. Contrairement à HLS et DASH, MP4 est livré sous la forme d'un seul fichier plutôt que de segments. Il est donc souvent utilisé dans les cas où vous souhaitez que les spectateurs puissent télécharger la vidéo.

R

REST
Transfert d'état représentatif : style architectural logiciel qui définit un ensemble de contraintes à utiliser pour créer des services Web. Les services Web conformes au style architectural REST, appelés services Web RESTful, assurent l'interopérabilité entre les systèmes informatiques sur Internet. Les services Web RESTful permettent aux systèmes demandeurs d'accéder aux représentations textuelles des ressources Web et de les manipuler en utilisant un ensemble uniforme et prédéfini d'opérations sans état. Brightcove utilise des API RESTful pour accéder à différentes parties de la plate-forme.
RTMP
Protocole de messagerie en temps réel : initialement un protocole propriétaire développé par Macromedia pour diffuser de l'audio, de la vidéo et des données sur Internet, entre un lecteur Flash et un serveur. Macromedia appartient maintenant à Adobe, qui a publié une version incomplète de la spécification du protocole à l'usage public.
